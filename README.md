# 29ver Audio Forensic Detector

Advanced Audio Analysis, Stem Separation, and Acoustic Classification System.

## 🚀 Overview

The **Audio Forensic Detector** is a full-stack Next.js and Python application designed to provide comprehensive forensic analysis of audio files. It leverages cutting-edge machine learning models to isolate individual tracks (like vocals, drums, background noise, or vehicles) and classify a wide range of acoustic events based on their spectral signatures.

## ⭐ Key Features

- **🔊 AI Source Separation:** Uses Meta's high-fidelity `htdemucs` transformer model to split audio into multi-stems (vocals, drums, bass, and environment/other).
- **🧠 Acoustic Event Classification:** Powered by YAMNet (TensorFlow Lite), it classifies environmental sounds (e.g., human voice, car engines, animals, sirens, gunshots) with confidence scoring.
- **📈 Spectral Analysis:** Real-time generation of frequency spectrums and acoustic profiling (RMS energy, dominant frequency, duration, decibel max limits).
- **🎙️ Live Audio Input:** Real-time audio recording and local file upload support (MP3, WAV, M4A, OGG).
- **📊 Interactive UI:** Modern dashboards displaying Sonar views, audio playback for individual stems, and event timelines.
- **📑 Forensic PDFs:** Generate downloadable forensic analysis reports (jsPDF).
- **🤖 Fast Batch Processing:** Multi-file inference capabilities for analyzing large datasets of audio at once.

## 🛠️ Technology Stack

- **Frontend Core:** Next.js 14+ (Webpack), React 18, TypeScript
- **Styling UI:** Tailwind CSS v3, Radix UI Components (`input-otp`, Accordions, Sliders), `lucide-react`
- **Audio Rendering:** Web Audio API, `wavesurfer.js`
- **Backend / Engine:** Python 3, FastAPI / NextJS Server Actions
- **Machine Learning Config:** PyTorch (`torchaudio`), TensorFlow Lite (`yamnet.tflite`), Demucs, Librosa, MediaPipe
- **Database Backend:** Supabase (PostgreSQL)

## 💻 Local Development Setup

To run this project locally, you must set up both the Node.js frontend and the Python forensic backend.

### 1. Requirements
- Node.js (v18+)
- Python (v3.8 - v3.11)
- FFmpeg (Must be installed and added to standard system PATH)

### 2. Install Python Dependencies
The custom Python machine learning scripts run inside the `Server/` and `scripts/` directories.
```bash
pip install -r requirements.txt
# Ensure you install Demucs separately if not in requirements:
pip install demucs librosa numpy torchaudio
```

### 3. Install Node.js Dependencies
```bash
npm install
```

### 4. Configuration
Create a `.env.local` file in the root directory and add your Supabase credentials:
```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
```

### 5. Start the Application
```bash
# Start the Next.js development server (using Webpack bundler)
npm run dev
```
Open [http://localhost:3000](http://localhost:3000)

## 🚢 Next.js Production Build Notes
This project requires CSS processing via PostCSS `autoprefixer` and Tailwind v3 configuration. It is built using Webpack instead of Turbopack to prevent CSS parsing failures generated by dependency utility classes.

To build for production:
```bash
npm run build 
```
